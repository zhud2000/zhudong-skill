# 会议纪要自动生成

## 用法

用户提供音频文件路径（支持 mp3/wav/m4a），一键生成结构化纪要。

执行方式：`/meeting-minutes <音频文件路径>`

## 工作流

```
音频文件 → Whisper 本地转写(transcribe.py) → 读取转写文本 → 按模板生成纪要 → 保存到 会议纪要/
```

## 执行步骤

### Step 1: 运行转写

```bash
python .claude/skills/meeting-minutes/transcribe.py <音频文件路径>
```

提取 `===TRANSCRIPT_START===` 和 `===TRANSCRIPT_END===` 之间的文本作为 `transcript`。

### Step 2: 读取模板

读取 `研究院工作区/会议纪要/会议纪要_模板.md` 获取纪要格式。

### Step 3: 生成纪要

基于转写文本，按模板格式生成结构化会议纪要。要求：

- **提取决策事项**：从讨论中识别"我们决定…"、"定了…"等决策信号，填入决策事项表
- **提取待办**：从讨论中识别"谁去…"、"下一步…"等行动信号，填入待办事项表，每项必须指定负责人
- **不要编造**：转写文本里没提到的不要添加
- **保留关键数字、术语、人名**：原样保留，不要改写
- **讨论要点按议题分组**：同一议题的所有观点放在一起
- **所有日期用 YYYY-MM-DD 格式**
- **文件名**: `会议纪要_YYYYMMDD_会议主题关键字.md`

### Step 4: 保存

保存到 `C:/Users/zhud2000/Desktop/工作文件/研究院工作区/会议纪要/` 目录。

### Step 5: 告知用户

告知用户纪要文件路径，并列出本次提取到的决策事项和待办事项各几条。
